SkyData es fácil

SkyData está diseñado pensando en desarrolladores que requieren productividad. El modelo es sencillo y completamente intuitivo. Se utiliza lo mejor de MVC, lo mejor de la orientación a servicios y lo mejor de un sistema gestionado por eventos.

Veamos un servicio (/SkyData/Core/Service/Controller/SkyDataServiceController) de una clase cualquiera:

	
class CochesServiceController extends /SkyData/Core/Service/Controller/SkyDataServiceController
{
			...
  /** 	
   * @ajaxMethod 
   * @runOnLoad
  **/ 	
  public function GetModelos ()
  {
	$result = array();
	$result[] = $this->NewDataRow ('Modelos', array ('id' => 1, 'descripcion' => 'Nissan'));
	$result[] = $this->NewDataRow ('Modelos', array ('id' => 2, 'descripcion' => 'Touran'));
	$result[] = $this->NewDataRow ('Modelos', array ('id' => 3, 'descripcion' => 'Audi'));
	$result[] = $this->NewDataRow ('Modelos', array ('id' => 4, 'descripcion' => 'Volkswagen'));
	
	return $result;
  }
		...
}

Y ahora el código de la plantilla Twig en la página (/SkyData/Core/Page/SkyDataPage) :


			...
<select  id='modelos' name="modeloSelected" ng-model="Modelos" class="span6">
	<option value="-1" selected='selected'>-- Elija un modelo --</option>
	<option ng-repeat="modelo in Modelos" value="[[modelo.id]]">[[modelo.descripcion]]</option>
</select>
			...

Voilá! El framework genera todo el código javascript necesario para que Angular haga su magia. El script auto generado utiliza $http para hacer la solicitud al servicio en el framework, genera la actualización de las variables, crea los controllers de Angular, etc.

Realmente no es todo: en el directorio de la clase de la página se coloca este fichero YAML (Sí!, se usa YAML también!): 

		
services:
  use: [CochesService]

Y en el directorio de la clase del servicio indicamos el modelo de datos:


datamodel:
  Modelos: [id, descripcion]
